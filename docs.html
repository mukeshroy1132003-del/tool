Quickstart

Get your first monitor running in under two minutes.

curl -X POST https://api.pulsegrid.io/v1/monitors \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Marketing site",
    "type": "https",
    "url": "https://example.com",
    "interval": 60
  }'

A successful response returns a monitor id you can use to fetch status, edit settings, or attach an alert policy.

Core concepts

Monitor — a single thing being checked (a URL, server, or API endpoint). Check — one execution of a monitor from one location. Incident — a period where a monitor is failing, grouped automatically from consecutive failed checks. Alert policy — the rules that decide who gets notified, and how.

Authentication

All API requests use a bearer token from your account's API Keys page. Keys can be scoped read-only or full-access, and rotated at any time without downtime.

Create a monitor

Monitors can be created from the dashboard, the API, or via Terraform using the official PulseGrid provider — useful for teams that manage monitoring as code alongside their infrastructure.

Check intervals

Intervals range from 30 seconds to 24 hours depending on your plan. Shorter intervals catch shorter outages, but also mean more checks counted against your plan's usage.

Global locations

Choose specific check locations per monitor, or let PulseGrid round-robin across all available regions to average out regional network noise.

Alert policies

An alert policy defines escalation steps: who's notified first, how long to wait before escalating, and which channel to use at each step.

{
  "name": "Production critical",
  "steps": [
    { "wait_minutes": 0, "notify": ["oncall-primary"], "channel": "sms" },
    { "wait_minutes": 5, "notify": ["oncall-secondary"], "channel": "voice" },
    { "wait_minutes": 15, "notify": ["#incidents"], "channel": "slack" }
  ]
}

Webhooks

PulseGrid can POST a JSON payload to any URL when a monitor changes state — useful for routing incidents into PagerDuty, Jira, or an internal tool.

REST API

The full REST API covers monitors, incidents, status pages, and account settings. Every endpoint returns predictable, versioned JSON, and rate limits scale with your plan.
